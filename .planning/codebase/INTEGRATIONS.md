# External Integrations

**Analysis Date:** 2026-01-26

## APIs & External Services

**Email Delivery:**
- Resend - Email service provider for transactional and template-based emails
  - SDK/Client: `resend` 4.1.0
  - Auth: `RESEND_API_KEY` environment variable
  - Implementation: `packages/api/src/funcs/emails/emails.funcs.ts` (getResendClient, sendEmail procedures)
  - From address: `noreply@resend.trueupcompanies.com`
  - Dev mode: Redirects emails to `delivered@resend.dev` to prevent test emails reaching real users

**Analytics & Monitoring:**
- PostHog - Event analytics and monitoring
  - Frontend: `posthog-js` 1.264.2 (via `@posthog/nextjs-config` 1.3.0)
  - Backend: `posthog-node` 5.22.0
  - Auth: `NEXT_PUBLIC_POSTHOG_KEY` public key
  - Host: `NEXT_PUBLIC_POSTHOG_HOST` (defaults to `https://us.i.posthog.com`)
  - Implementation:
    - Frontend: Built into Next.js config
    - Backend: `packages/api/src/utils/posthog-server.ts` (getPostHogServer, captureServerException)
    - Disabled in development, enabled in production only
    - Server-side errors captured with context (userId, path, errorCode, extra data)

**Background Jobs & Task Scheduling:**
- Trigger.dev (v4) - Asynchronous task execution and cron scheduling
  - SDK: `@trigger.dev/sdk` 4.0.4
  - Location: `packages/tasks/src/trigger/`
  - Key tasks defined:
    - `ticketsNotification` - Async job queue processor with retry logic
    - `recover-queued-jobs` - Cron task for job recovery
    - `testJob` - Test/example task
  - Handler pattern: `packages/tasks/src/trigger/handlers/tickets.ts`
  - Features used: Task retry configuration, metadata tracking, logging, tags
  - No environment variable required (configured in Trigger.dev dashboard)

## Data Storage

**Primary Database:**
- PostgreSQL 17 (via Supabase)
  - Connection: `POSTGRES_URL` environment variable
  - Client options: `pg` 8.12.0, `postgres` 3.4.4, `@vercel/postgres` 0.9.0
  - ORM: Drizzle ORM with automatic schema migration
  - Schema location: `packages/db/src/schema/` (table definitions with `*.table.ts` suffix)
  - Migrations: `supabase/migrations/` (auto-generated by Drizzle Kit)
  - Local development: Supabase CLI with local PostgreSQL on port 54322

**File Storage:**
- Supabase Storage - Cloud file storage via Supabase
  - Client: `@supabase/supabase-js` 2.90.1
  - Implementation: `packages/api/src/funcs/hr/storage.funcs.ts` (uploadToStorage, getStorageURL)
  - Use cases: HR documents, attachments, file uploads
  - Signed URLs: Generated for secure file access
  - No TTL configured by default

**Caching:**
- Server-side caching via React Query - Memory-based, no external service
  - Library: `@tanstack/react-query` 5.50.0
  - Configured in: `apps/web/src/app/providers/`
  - Uses tRPC integration for automatic cache management

## Authentication & Identity

**Auth Provider:**
- Supabase Auth - Managed authentication with phone/email OTP and password auth
  - Client: `@supabase/supabase-js` 2.90.1
  - SSR support: `@supabase/ssr` 0.8.0
  - Server-side client: `packages/api/src/utils/supabase/server.ts` (createSBAdminServer)
  - Implementation:
    - tRPC context obtains user from Supabase JWT token claims
    - Automatic user ID extraction from auth token in `packages/api/src/trpc.ts`
    - Row-level security (RLS) policies likely used in Supabase
  - Protected procedures: Available via `protectedProcedure` in tRPC routers
  - Public procedures: Available via `publicProcedure` for login/registration

**Authorization:**
- CASL (Access Control Service Layer)
  - Library: `@casl/ability` 6.7.3, `@casl/react` 5.0.0
  - Implementation: Role-based access control (RBAC) throughout UI
  - Permission checks in site navigation config: `apps/web/src/core/config/site.ts`
  - Custom permissions system likely defined in `packages/db/src/acl/`

## Monitoring & Observability

**Error Tracking:**
- PostHog (limited) - Captures server exceptions only

  - Context captured: userId, path, errorCode, exception type, stack trace

**Logs:**
- Console logging for development

- Production: PostHog for analytics, application logs typically handled by deployment platform

## Environment Configuration

**Required env vars:**
- `NEXT_PUBLIC_SUPABASE_URL` - Supabase project URL (public)
- `NEXT_PUBLIC_SUPABASE_ANON_KEY` - Supabase anonymous key (public)
- `SUPABASE_SERVICE_ROLE_KEY` - Supabase admin key (secret)
- `POSTGRES_URL` - PostgreSQL connection string
- `RESEND_API_KEY` - Resend email service key

**Optional env vars:**
- `NEXT_PUBLIC_POSTHOG_KEY` - PostHog project key
- `NEXT_PUBLIC_POSTHOG_HOST` - PostHog instance URL (custom domain)
- `NEXT_PUBLIC_POSTHOG_PROJECT_ID` - PostHog project ID
- `SECRET_CLIENT_COOKIE_VAR` - Client-side cookie encryption key (JSON Web Key format)
- `SUPABASE_DB_PASSWORD` - Local Supabase database password
- `SUPABASE_PROJECT_ID` - Supabase project identifier
- `NODE_ENV` - Runtime environment (development/production)

**Secrets location:**
- `.env` file (local development only, not committed)
- Vercel dashboard (production secrets)
- Environment variables passed at runtime via deployment platform
