<objective>
Update mobile app dependencies to latest Expo SDK 54 patch versions and configure monorepo hoisting for React Native compatibility.

Purpose: The mobile app runs Expo SDK 54 (~54.0.22) which needs a patch update to ~54.0.31 for full iOS 26 compatibility. Output: Updated package.json with latest SDK 54 patches, properly configured .npmrc, clean expo-doctor output.

<task type="auto">
  <name>Task 1: Configure monorepo hoisting and update Expo dependencies</name>
  <files>.npmrc, apps/mobile/package.json</files>
  <action>
1. Write `node-linker=hoisted` to the root `.npmrc` file. This is MANDATORY for React Native in pnpm monorepos -- Metro bundler expects a flat node_modules structure and will fail with "Unable to resolve module" errors without hoisting.

2. From the `apps/mobile` directory, run:
   ```
   npx expo install expo@latest expo-router@latest --fix
   ```
   This updates expo (~54.0.22 -> ~54.0.31), expo-router (^6.0.14 -> ^6.0.21), and auto-fixes any peer dependency mismatches within SDK 54.

3. Run `pnpm install` from the repo root to reinstall with hoisted node_modules.

4. Run `npx expo-doctor` from apps/mobile to validate all dependencies are compatible. Fix any issues it reports.

IMPORTANT: Stay on SDK 54 -- do NOT upgrade to SDK 55 beta. SDK 55 drops Legacy Architecture support and has breaking changes not suitable for this phase.

If expo-doctor warns about statusBar in app.json -- this is already handled correctly (the app uses expo-status-bar package, and app.json does NOT have a statusBar field).
  </action>
  <verify>
Run from apps/mobile directory:
- `npx expo-doctor` exits with 0 or only non-critical warnings
- `cat ../../.npmrc` shows `node-linker=hoisted`
- `grep '"expo":' package.json` shows ~54.0.31 or similar latest SDK 54 patch
- `grep '"expo-router":' package.json` shows updated version
  </verify>
  <done>
- .npmrc contains node-linker=hoisted
- expo updated to latest SDK 54 patch (>=54.0.31)
- expo-router updated to latest v6 patch
- expo-doctor reports no critical issues
- pnpm install completes without errors
  </done>
