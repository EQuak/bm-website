  <action>
1. **Create server component** at `developer-tools/page.tsx`:
   - Follow existing settings page pattern (look at `user-roles/[...mode]/page.tsx` or similar for reference)
   - Call `await api.profiles.getRealProfile()` to get the real user's profile
   - Gate access: if `profile.aclRole !== "developer"`, redirect to `/settings` using `redirect()` from `next/navigation`
   - Prefetch `api.profiles.getAllProfilesAlphabetically.prefetch()` for the combobox
   - Wrap `<DeveloperToolsContent realUserId={realProfile.user_id} />` in `<HydrateClient>`

2. **Create client component** at `developer-tools/_components/DeveloperToolsContent.tsx`:
   - `"use client"` directive
   - Props: `{ realUserId: string }`
   - Use `api.profiles.getAllProfilesAlphabetically.useSuspenseQuery()` to get all profiles
   - Read current cookie state with `Cookies.get("dev_impersonation")` from `js-cookie`
   - Build a Mantine `Select` component with `searchable` prop:
     - Data: map profiles to `{ value: profile.id, label: "firstName lastName — roleName — department" }`
     - Placeholder: "Select a profile to impersonate"
     - When a profile is selected: `Cookies.set("dev_impersonation", \`${realUserId}:${profileId}\`, { expires: 1, sameSite: "strict", path: "/" })` then `router.refresh()`
   - If currently impersonating, show a "Stop Impersonating" button (red, `onClick` → `Cookies.remove("dev_impersonation", { path: "/" })` then `router.refresh()`)
   - Show current impersonation status: if active, display the impersonated profile name in a red Alert/Badge
   - Use `useRouter()` from `next/navigation` for `router.refresh()`
   - Cookie constants: name `"dev_impersonation"`, expiry 1 day

3. **Add nav link** in `site.ts`:
   - Find the settings navigation links array
   - Add a new entry for "Developer Tools" with `href: "/developer-tools"`
   - Use the same permissions pattern as other settings links (the page itself handles developer role gating)
   - Use an appropriate icon (e.g., `IconCode` or `IconTool` from tabler icons)
  </action>
