# `docs/` — template and product intent

Files here are **repo-level** documentation: how to use the starter and what *your fork* is trying to achieve.

| File | Purpose |
| ---- | ------- |
| [`TEMPLATE.md`](TEMPLATE.md) | What this repository is (and is not), fork notes, mobile/CI caveats |
| [`PROJECT_INTENT.md`](PROJECT_INTENT.md) | **Your product:** goals, users, non-goals—read by contributors and AI for scope |

**Why not `ai/docs/`?** [`../ai/docs/`](../ai/docs/) is **technical** reference (API shape, Drizzle patterns, UI conventions). [`PROJECT_INTENT.md`](PROJECT_INTENT.md) belongs next to [`TEMPLATE.md`](TEMPLATE.md) so onboarding and product context stay in one place. It is still listed in `CLAUDE.md` for agents.
